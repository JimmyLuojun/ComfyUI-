# 学习路线

从跑通环境到内容创作应用，分五个阶段推进。每个阶段完成后在 README.md 的进度 checklist 中更新。

---

## 阶段一：跑通环境

**目标：** 能生成第一张图，确认 MPS 加速正常。

**完成标准：**
- [ ] ComfyUI 启动，终端显示 `Device: mps`
- [ ] 成功加载一个 SDXL 模型
- [ ] 生成第一张图并保存

**学习重点：**
- ComfyUI 界面布局和节点连接逻辑
- 核心节点：Load Checkpoint、CLIP Text Encode、KSampler、VAE Decode、Save Image
- `extra_model_paths.yaml` 配置

**参考资源：**
- ComfyUI 官方 GitHub：https://github.com/comfyanonymous/ComfyUI
- ComfyUI 官方 Wiki

---

## 阶段二：掌握 SDXL 基础出图

**目标：** 熟悉 SDXL 出图流程，形成自己的基础参数模板。

**完成标准：**
- [ ] 测试 2-3 个 SDXL 写实模型
- [ ] 理解 Prompt、CFG、Steps、Seed、分辨率的作用
- [ ] 保存第一个可复用的文生图工作流
- [ ] 填写 2-3 条模型记录到 model_log.md

**学习重点：**
- 正向/负向提示词结构
- 采样器选择（DPM++ 2M Karras 入门首选）
- SDXL 推荐分辨率（1024x1024 / 832x1216）

---

## 阶段三：LoRA 与风格控制

**目标：** 学会叠加 LoRA，能控制人物风格和场景风格。

**完成标准：**
- [ ] 成功加载并测试 2 个 LoRA
- [ ] 理解 LoRA 权重对效果的影响
- [ ] 记录 LoRA 和基础模型的兼容性测试结果

**学习重点：**
- Load LoRA 节点的使用
- LoRA 权重调节（0.5-1.0 范围内测试）
- 多 LoRA 叠加技巧
- 模型与 LoRA 的兼容性判断

---

## 阶段四：尝试 Flux GGUF

**目标：** 跑通 Flux 工作流，评估 Mac 上的实际表现。

**完成标准：**
- [ ] 成功用 Flux.1 schnell GGUF 生成图片
- [ ] 对比 Flux 和 SDXL 的效果、速度、内存占用
- [ ] 记录 Mac M4 Pro 24GB 上可用的 Flux 量化版本

**学习重点：**
- Flux 工作流结构（与 SDXL 的节点差异）
- GGUF 量化版本选择（Q4_K_S / Q5_K_S / Q6_K）
- Flux 的文本理解能力特点
- Mac MPS 环境下的已知限制

---

## 阶段五：内容创作应用

**目标：** 将工作流应用到实际内容创作，形成可复用的生产流程。

**完成标准：**
- [ ] 能稳定生成课程封面图
- [ ] 能稳定生成人物角色图
- [ ] 形成 3-5 个带提示词模板的可复用工作流
- [ ] 建立商业授权确认流程

**应用方向：**
- AI 课程封面、小红书封面、视频号封面
- 产品场景图、角色设定图
- 漫剧分镜素材
- 短视频配图、直播演示案例

---

- 将 ComfyUI 融入 AI 内容生产流程
- 形成可教学、可演示的 AIGC 工作流体系
